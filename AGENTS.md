# AGENTS.md

## 项目名称
阿木作战台，英文项目名 amu-battle-station。

## 项目目标
这是一个个人每日作战台网页 App，用于管理每日行动，包括今日任务、闲鱼运营、身体打卡、资金纪律、每日复盘和督促提醒。

## 技术栈
React + Vite + JavaScript + Tailwind CSS。
默认使用 localStorage 保存数据。
JSON 导入导出作为主要备份方式。

## 产品原则
优先简单可用。
优先桌面网页端使用。
手机端只保证基本可访问，不作为主优化场景。
优先每天真实会打开。
不要堆功能。
不要做成企业后台。
不要做成复杂数据库。
不要做成飞书表格风。
目标是个人作战仪表盘。

## 范围边界
不做后端。
不做登录注册。
不接飞书 API。
不做客户交付台。
不接外部 API。
云同步代码可以保留，但 UI 默认隐藏，不主动启用。

## 页面范围
保留 6 个页面：
1. 今日作战台
2. 闲鱼运营台
3. 身体打卡台
4. 资金雷达台
5. 每日复盘台
6. 督促页

## 设计风格
参考目标图：
左侧深色导航。
右侧浅色内容区。
顶部核心状态卡片。
中部左侧今日任务清单。
中部右侧风险提醒、运营概览、身体概览、资金雷达概览。
整体简洁、克制、有战斗感。
不要大面积复杂渐变。
不要花哨动效。
不要把页面做成普通表格。

## 公开仓库规则
不允许把实际资金金额写进默认代码或文档。
不允许把真实客户资料写进仓库。
不允许提交 .env、API key、token、secret。
文档中的资产、账号、金额只能使用演示数据。
默认资产只能使用演示资产。

## 身体页规则
自律状态属于私密记录字段。
必须支持公开展示模式隐藏相关 UI。
隐藏只影响 UI，不影响数据保存和 JSON 导出。
完整记录模式用于个人自查。

## 督促页规则
督促页只提醒养号、学习、复盘、睡觉。
起床只做目标追踪，不作为网页提醒卡片。
网页提醒只有页面打开时生效，强提醒请配合手机或电脑闹钟。

## 开发规则
修改前先检查当前代码结构。
尽量小步修改。
不要一次性重构全部代码。
不要删除已有功能。
新增依赖前先说明原因。
修改后运行 npm run dev 检查。
如果有 lint 或 build 命令，也要运行。
如果不确定，先读 HANDOFF.md 和 TODO.md。
