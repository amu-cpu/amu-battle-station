# AGENTS.md

## 项目名称
阿木作战台，英文项目名 amu-battle-station。

## 项目目标
这是一个个人每日作战台网页 App，用于管理我的每日行动，包括今日任务、闲鱼代写运营、身体打卡、资金纪律和每日复盘。

## 技术栈
React + Vite + JavaScript + Tailwind CSS。
第一版使用 localStorage 保存数据。
不做后端。
不做登录注册。
不接外部 API。
不做客户交付台。
不接飞书 API。

## 产品原则
优先简单可用。
优先手机端可用。
优先每天真实会打开。
不要堆功能。
不要做成企业后台。
不要做成复杂数据库。
不要做成飞书表格风。
目标是个人作战仪表盘。

## 页面范围
只保留 5 个核心页面：
1. 今日作战台
2. 闲鱼运营台
3. 身体打卡台
4. 资金雷达台
5. 每日复盘台

## 设计风格
参考目标图：
左侧深色导航。
右侧浅色内容区。
顶部 5 个核心状态卡片。
中部左侧今日任务清单。
中部右侧风险提醒、运营概览、身体概览、资金雷达概览。
整体简洁、克制、有战斗感。
不要大面积复杂渐变。
不要花哨动效。
不要把页面做成普通表格。

## 开发规则
修改前先检查当前代码结构。
尽量小步修改。
不要一次性重构全部代码。
不要删除已有功能。
新增依赖前先说明原因。
修改后运行 npm run dev 检查。
如果有 lint 或 build 命令，也要运行。
如果不确定，先读 HANDOFF.md 和 TODO.md。
