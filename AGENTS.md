# AGENTS.md

## 项目名称
阿木作战台，英文项目名 amu-battle-station。

## 项目目标
这是一个个人每日作战台网页 App，用于管理每日行动，包括今日任务、闲鱼运营、身体打卡、资金纪律、每日复盘和督促提醒。

## 技术栈
React + Vite + JavaScript + Tailwind CSS。
默认使用 localStorage 保存数据。
JSON 导入导出作为主要备份方式。

## 产品原则
优先简单可用。
优先桌面网页端使用。
手机端只保证基本可访问，不作为主优化场景。
优先每天真实会打开。
不要堆功能。
不要做成企业后台。
不要做成复杂数据库。
不要做成飞书表格风。
目标是个人作战仪表盘。
复盘页负责决定明天打什么仗。
今日页负责当天执行。
今日作战台是仪表盘，不是长表单页。
今日作战台必须优先利用桌面宽度。
今日页应该优先一屏展示今日状态。
首页默认不应出现多个内部滚动条。
首页只展示关键摘要。
今日页只做概览和关键动作。
完整明细通过展开或对应页面查看。
详情操作应放在各独立页面。
不允许为了“一屏显示”把内容缩成很小。
不允许把整页滚动问题转移成卡片内部小滚轮问题。
首页只放当天执行动作。
已经内化的纪律不要反复作为任务出现。
资金模块是查看和纪律管理，不再是每日默认任务。
分类摘要卡片应支持点击筛选。
首页资金金额必须可隐藏，方便截图。
“明天最重要 3 件事”必须能转化为第二天的今日重点任务。
学习任务和今日重点任务不能混为一谈。
新增联动必须保护旧 localStorage 数据。
自动生成任务必须防止重复生成。
新生成默认任务不再包含资金类默认任务。
不清空 localStorage。
不强制删除旧资金任务历史。

## 范围边界
不做后端。
不做登录注册。
不接飞书 API。
不做客户交付台。
不接外部 API。
云同步代码可以保留，但 UI 默认隐藏，不主动启用。

## 页面范围
保留 6 个页面：
1. 今日作战台
2. 闲鱼运营台
3. 身体打卡台
4. 资金雷达台
5. 每日复盘台
6. 督促页

## 设计风格
参考目标图：
左侧深色导航。
右侧浅色内容区。
顶部核心状态卡片。
中部左侧今日任务清单。
中部右侧风险提醒、运营概览、身体概览、资金雷达概览。
整体简洁、克制、有战斗感。
不要大面积复杂渐变。
不要花哨动效。
不要把页面做成普通表格。

## 公开仓库规则
不允许把实际资金金额写进默认代码或文档。
不允许把真实客户资料写进仓库。
不允许提交 .env、API key、token、secret。
文档中的资产、账号、金额只能使用演示数据。
默认资产只能使用演示资产。

## 身体页规则
自律状态属于私密记录字段。
UI 使用中性表达，不使用直白敏感词。
必须支持公开展示模式隐藏相关 UI。
隐藏只影响 UI，不影响数据保存和 JSON 导出。
完整记录模式用于个人自查。
身体页只保留“展示模式：完整”和“展示模式：公开”两种展示模式。
历史记录只能叫“展开完整历史 / 收起完整历史”，不允许当作展示模式命名。
连续天数和距上次失守天数用于提醒，不用于羞辱用户。
急救模式优先引导延迟和替代动作。
急救模式必须是流程，不是按钮堆。
自律备注、触发源默认不强制填写。
奖励建议不能长期平铺展示，应按里程碑出现。
自律功能强调行动干预，不做失败审判。
身体页三栏布局要统一视觉节奏。
公开展示模式必须自然隐藏自律相关内容。
不允许清空旧 bodyRecords。
不允许删除旧 relapseStatus 数据。

## 督促页规则
督促页只提醒养号、学习、复盘、睡前收尾。
起床只做目标追踪，不作为网页提醒卡片。
网页提醒只有页面打开时生效，强提醒请配合手机或电脑闹钟。
督促提醒次数只统计系统真实触发次数。
点击“稍后提醒”不能增加提醒次数。
不允许 UI 重渲染导致重复提醒。
睡前提醒统一叫“睡前收尾”。

## 开发规则
修改前先检查当前代码结构。
尽量小步修改。
不要一次性重构全部代码。
不要删除已有功能。
新增依赖前先说明原因。
修改后运行 npm run dev 检查。
如果有 lint 或 build 命令，也要运行。
如果不确定，先读 HANDOFF.md 和 TODO.md。

## 身体页体验精修补充

1. 急救模式必须突出当前步骤，不允许按钮堆。
2. 身体页以今日身体记录为主体，自律模块为辅助。
3. 自律功能强调行动干预，不做失败审判。
4. 可选备注默认折叠。
5. 历史记录默认摘要化，完整表格可展开。
6. 奖励建议只按里程碑出现。
7. 不允许清空旧 bodyRecords。
8. 不允许引入 react-hooks/set-state-in-effect 报错。
9. 不允许混淆展示模式和历史展开。
10. 身体页不新增无意义模式。
11. 急救模式要突出下一步行动。
12. 视觉收口优先，不再新增功能。
